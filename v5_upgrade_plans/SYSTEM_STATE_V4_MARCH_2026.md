## 2. Foundational Pillars

### 2.1 Hierarchical Cognitive Memory (H-MEM)
The system's "Brain" is no longer stateless. It uses a 3-tier memory model:
-   **Episodic Memory (`episodic_traces`):** Immutable logs of every interaction (emails, transcripts, calls).
-   **Semantic Memory (`semantic_facts`):** A distilled "Knowledge Graph" of seller traits and deal truths.
-   **Bayesian Belief Updating:** Conflicting signals from sellers are resolved mathematically (Alpha/Beta parameters), allowing the system to maintain a high-fidelity "Confidence Score" in its own strategic assumptions.

### 2.2 Early-Fusion Sensory Architecture
The system "Sees" through an integrated multimodal gateway (`llm.py`):
-   **Native Reasoning:** Unlike legacy OCR systems, the v4.0 engine ingests image tokens and text tokens into the same latent space.
-   **Strategic Visuals:** The system cross-references building photos (physical decay) against tax records (assessed condition) to find leverage.

### 2.3 Generative UI (The Dynamic Cockpit)
The dashboard is no longer a static grid. It is a context-aware engine:
-   **Self-Assembly:** The UI dynamically renders specialized engineering tools (e.g., Debt-Wrap Simulators, Site-Plan Calculators) based on the Manager's strategic verdict.
-   **State Persistence:** Powered by a hard-initialized LangGraph checkpointer, the Workspace maintains Roman's deal context across all dashboard pages.
